# 좌표계 합의 회신 — 논의 필요-12 (SYS-BOARD §7)

> 기획 문서(SYS-BOARD §7)가 "행/열 부호, 회전 방향(시계/반시계), 행렬 기준은 구현 착수 전
> 프로그래밍팀과 확정"이라 요청한 항목에 대한 **현행 구현 실측 회신**입니다.
> 작성: 캐릭터/스킬 담당 · 2026-07-12 · 근거 코드는 각 절에 명기.

## 1. 결론 요약 (기획 잠정안과의 차이)

| 항목 | 기획 잠정안 (SYS-BOARD §7) | 현행 구현 | 판정 |
|---|---|---|---|
| 표기 | (행, 열) — 좌상단 (0,0) | **(col, row) — 좌하단 (0,0)** | **차이 있음 — 합의 필요** |
| 세로 부호 | 아래로 증가 (화면 좌표계) | **row+ = 월드 위(UP)** (수학 좌표계) | **차이 있음 — 합의 필요** |
| 기준 방향 | 위(UP) — 범위 마스크 작성 기준 | 위(UP) — 동일 | 일치 |
| 회전 | 0/90/180/270° (방향 종속만) | 동일 — **시계방향** k회, `(dc,dr) → (dr,−dc)` | 일치 (방향 명시 확정) |

## 2. 현행 구현 상세

### 2-1. 칸 주소
- 표기: `(col, row)` 순서. 문자열 키는 `"col_row"` (예: `"3_5"`).
- 원점: **좌하단 (0,0)**. `col+` = 오른쪽, `row+` = **위쪽**.
- 근거: `BoardService.CellToWorld`가 row 증가를 월드 y+(위)로 투영. 보드 계약(2026-07-04 합의)
  "좌표 (col,row), row+ = world up" 계승.

### 2-2. 방향 벡터 (facing)
```
UP    = ( dc=0,  dr=+1 )
RIGHT = ( dc=+1, dr=0  )
DOWN  = ( dc=0,  dr=−1 )
LEFT  = ( dc=−1, dr=0  )
```
근거: `CombatResolveLogic.FacingVector` (CombatResolveLogic.mlua:21-29).

### 2-3. 범위 마스크와 회전
- 범위 패턴 마스크는 **UP 기준으로 작성** (기획 잠정안과 동일 — 시트 가이드 §4-1 계승).
- 방향 종속(`dirDependent=true`) 스킬만 발동 방향으로 회전. 방향 무관은 회전 없음.
- 회전 공식: **시계방향 90° k회**, `(dc, dr) → (dr, −dc)`
  - UP→RIGHT k=1, UP→DOWN k=2, UP→LEFT k=3
- 근거: `CombatResolveLogic.RotateOffset` (CombatResolveLogic.mlua:31-47).

### 2-4. 거리
- **맨해튼 거리** `|Δcol| + |Δrow|` (SYS-COMBAT §4 제안과 일치). 동률은 랜덤 타이브레이크.
- 근거: `CombatResolveLogic.SelectTargets`.

## 3. 합의 요청 사항

1. **(행,열)·좌상단 원점(기획 표기) vs (col,row)·좌하단 원점(구현)** 중 하나로 통일 필요.
   - 구현 변경 비용: 좌표계 반전은 보드 계약(BoardService/BoardCatalog/렌더링)까지 걸친
     전면 수정이므로 **문서 표기를 구현에 맞추는 쪽을 권장** — 게임플레이 규칙(범위 회전·
     거리·랜덤)은 어느 표기든 동치라 플레이어 체감 차이는 없음.
   - 기획 문서 쪽 수정 시: "표기 (col,row) · 좌하단 (0,0) · row+ = 위" 한 줄이면 됨.
2. 회전 방향은 **시계방향**으로 이미 구현·검증됨(E2E 통과) — 문서에 "시계"만 명기하면 확정.
3. 범위 마스크 작성 기준(UP)·맨해튼 거리·동률 랜덤은 기획 잠정안과 일치 — 그대로 확정 가능.

## 4. 참고 — 마스크 작성자 가이드 (기획·데이터 작업용)

UP 기준 마스크에서 `dr=+1`이 "캐릭터 앞쪽 1칸"입니다. 예: 직선 사거리 2 (`LINE_2`)
```
offsets = [ {dc=0, dr=1}, {dc=0, dr=2} ]
```
RIGHT 발동 시 자동 회전되어 `{1,0}, {2,0}`이 됩니다. 판 밖 칸은 버림(클램프 없음 — SYS-COMBAT §3-1 A안).
